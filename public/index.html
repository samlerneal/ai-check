原理

评分原理

出口地区

用 HTTP 出口国家、地区和 Anthropic Supported Countries allowlist 对比。不在清单或属于官方排除区域时直接满分扣分。

出口网络画像

先看出口国家是否支持，再看 ASN/AS 组织是否像共享代理、云厂商或中转网络。地区不支持优先于一致性。

IPv4 / IPv6

分别检查 IPv4 与 IPv6 出口。任一路径落在限制地区都扣分；两条路径国家不一致也扣分。

DNS 解析出口

DNS resolver 需要 authoritative DNS 日志或外部 DNS 事件源。未接入时不伪造结果；接入后按 resolver 国家与 HTTP 出口一致性评分。

浏览器语言与时区

浏览器语言或时区如果直接指向限制地区会扣分；否则才作为与网络出口的一致性辅助信号。

WebRTC

检查 WebRTC candidate 是否暴露可路由公网地址。它本身不能稳定判断国家，但可发现额外网络路径。

Claude Code BASE_URL

检查 ANTHROPIC_BASE_URL 是否指向官方 API 或明确授权路径。不明端点、其他模型供应商或限制地区相关域名会扣分。

代理变量

检查 HTTP_PROXY、HTTPS_PROXY、ALL_PROXY、GRPC_PROXY。代理越多路径越不清晰；限制地区相关代理直接高风险。

本机 Locale / shell 时区

本机语言和时区如果指向限制地区会扣分；同时检查它们是否与浏览器和网络出口冲突。

不评分项

中文输入法、普通中文软件、用途自报等网页无法可靠检测或缺乏明确公开条款依据，不进入评分。
